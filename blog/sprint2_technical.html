CSS Positioning

In CSS there is a property called position. There are a number of values associated with that property but there are three main ones that are used and are quite powerful.

As you can see we have a box which is a block level element that just flows vertically down a page in a normal flow.

Using positioning we can move the box around anywhere on the page.

I will start with the property of relative. We can use relative to fine tune where things are positioned. For example if we want to push a side bar outside the container but in such a way so it remains within the flow of the content.

We have access to four properties (top, left, bottom & right) with which we can manipulate a relatively positioned element. These properies are normally stated in pixel or percentage numeric values and are always calculated from the top left corner of the element.

relative

#relative {

position: relative;

top: 50px; /* Move element 50px down from top left corner */

left: -30px; /* Move element 10px right from the top left corner */

}

relative

Here the box with the id # relative is moved 50px down and 30 pixels right using the negative value to position the box -30px from the left.

Same effect can be achieved using right: 30px;

absolute

Absolute positioning is a bit trickier and kind of interesting. If I was to declare the box position property as absolute, it would take the box outside the flow of the parent element. It would be sitting outside the flow of the container signified by the box boundaries.

#absolute {

position: absolute;

}

absolute

!!!! the container does not even know that the box element exists, it is taken outside its scope.

That is the point because now we have more flexibility as to where we can move and position the box. However because there are no elements that are positioned that contain the box, the box will default to the <body> element when specifying the top and left coordinates. Like so:

#absolute {

position: absolute;

top: 0;

left: 0;

absolute

Now if I specify the container element, which does not have any specified position property, the position property of relative. The "ABSOLUTE" box would now sit in the top left corner of the container itself and not the <body> element.

This is because the container now has a position property and it is the parent of the box element, so the box element will now reference itself in relation to its parent container.

#container {

background: white;

padding: 20px;

postion: relative;

}

fixed

Fixed positioned elements essentially behave the same as absolute postioned elements. The elements are taken out of the flow but are fixed on the screen wherever you hard-code the position.

#fixed {

position: fixed;

right: 150px;

top: 150px;
